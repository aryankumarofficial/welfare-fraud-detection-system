# Database Population Matrix

This matrix documents which database tables are populated automatically, manually, or only via a seeder/runtime path.

| Table | Population Method | Automatic on startup? | Manual / Seeder / Runtime |
|---|---|---|---|
| `student_profiles` | Schema migration only; no rows inserted | ❌ | Manual via future import or demo seeder |
| `student_financial_records` | Schema migration only; no rows inserted | ❌ | Manual via future import |
| `student_social_records` | Schema migration only; no rows inserted | ❌ | Manual via future import |
| `student_transaction_summaries` | Schema migration only; no rows inserted | ❌ | Manual via future import |
| `student_medical_summaries` | Schema migration only; no rows inserted | ❌ | Manual via future import |
| `feature_snapshots` | Schema migration only; no rows inserted | ❌ | Runtime generated if source records exist |
| `prediction_records` | Schema migration only; no rows inserted | ❌ | Runtime generated if predictions are executed |
| `prediction_jobs` | Schema migration only; no rows inserted | ❌ | Runtime generated by queue/process flows |
| `prediction_reviews` | Schema migration only; no rows inserted | ❌ | Manual or runtime review flows |
| `model_versions` | Schema migration only; no rows inserted | ❌ | Manual or runtime model registry actions |
| `audit_logs` | Schema migration only; no rows inserted | ❌ | Runtime generated by audit/event flows |

## Notes

- The only automatic action on startup is database schema creation via `db:migrate`.
- There is no automatic population of source data or ML runtime tables from repository CSV data.
- The `services/ml/src/train.py` training flow stores model artifacts on disk, not in PostgreSQL.
- `scripts/seed-demo-data.ts` is a demo data generator and does not perform database writes.
